How it works

The full agent loop

Not a one-shot prompt. CODI plans, executes tools, verifies results, and synthesizes a response. Every time.

01 · Plan
Understands your task

Refines your prompt, indexes your codebase, and creates a step-by-step execution plan before touching any file.

02 · Execute
Calls real tools

Reads files, writes code, runs shell commands. Actual tool calls — not hallucinated output you have to copy manually.

03 · Verify
Checks its own work

After each execution, CODI checks whether the task was actually completed. If not, it retries with context from the failure.

04 · RAG
Knows your codebase

ChromaDB vector search over your entire project. CODI finds relevant code before editing — no hallucinated imports.

05 · Plugins
21+ MCP servers

Git, browser automation, PostgreSQL, Redis, Docker, GitHub — toggle any at runtime. No code changes needed.

06 · Providers
Any LLM, one line

Ollama locally, Groq for speed, Anthropic for quality, Gemini as fallback. Switch with /mode in the CLI.

Setup

Running in 5 minutes

Windows, macOS, Linux. Works fully offline with Ollama, or with a free Groq API key for cloud speed.

01 — Prerequisites by OS
Requires Python 3.10+, Node.js 18+, and Git. Install via python.org, nodejs.org, and git-scm.com — or use winget: 
# Run in PowerShell (as Administrator)
winget install Python.Python.3.12
winget install OpenJS.NodeJS.LTS
winget install Git.Git

# Optional: Ollama for local/offline mode
winget install Ollama.Ollama
Requires Python 3.10+, Node.js 18+, and Git (Git ships with Xcode CLT). Use Homebrew: 
# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

brew install python@3.12 node git

# Optional: Ollama for local/offline mode
brew install ollama
Requires Python 3.10+, pip, Node.js 18+, and Git. Commands below are for Debian/Ubuntu — adjust for your distro: 
# Debian / Ubuntu
sudo apt update
sudo apt install python3.12 python3.12-venv python3-pip git curl

# Node.js 20.x via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Optional: Ollama (any distro, installs via script)
curl -fsSL https://ollama.com/install.sh | sh
02 — Clone the repo
git clone https://github.com/AbheyTiwari/CODI
cd CODI/coding-agent
03 — Create a virtual environment
# Windows (PowerShell)
python -m venv codi
.\codi\Scripts\Activate.ps1

# macOS / Linux
python3 -m venv codi
source codi/bin/activate
04 — Install dependencies
pip install -r requirements.txt
05 — Register the global codi command
# Run once from inside coding-agent/
pip install -e .
Now you can type codi in any project directory and it auto-launches with that folder indexed. No more python main.py.
06 — Add your API key (optional for cloud/hybrid)
# Create a .env file in coding-agent/
CODI_GROQ_API_KEY=gsk_your_key_here

# Other providers (use whichever you have)
CODI_ANTHROPIC_API_KEY=sk-ant-...
CODI_OPENAI_API_KEY=sk-...
CODI_GEMINI_API_KEY=AIza...
07 — Navigate to any project and launch
# Works in any folder on your system
cd /path/to/your/project
codi

# CODI auto-indexes the folder and starts
Windows: codi not found in new terminals?
The codi command lives inside your venv. To make it work in every new terminal without activating the venv first, run this once in an admin PowerShell: 
# Replace the path with your actual venv Scripts folder
$p = "C:\path\to\CODI\coding-agent\codi\Scripts"
[Environment]::SetEnvironmentVariable(
  "PATH",
  [Environment]::GetEnvironmentVariable("PATH","User") + ";$p",
  "User"
)
# Restart all terminals after running this
Modes

Switch modes instantly at runtime with /mode <name> — no restart needed.

LOCAL
100% offline

Ollama only. No internet. No API costs. Your code never leaves your machine.

Needs: ollama pull phi3:mini + qwen2.5-coder:7b
HYBRID
Local first, cloud fallback

Ollama handles most tasks. Escalates to cloud only when context overflows or local is unavailable.

Recommended default
CLOUD
Maximum capability

Every call goes to your cloud provider. Groq is free and fast. Anthropic for best quality.

Needs: API key in .env
AIR
Phone inference

Runs a quantized model on your Android phone over Wi-Fi. Good middle ground — no GPU needed on PC.

AirLLM — phone inference

Run LLMs on your Android phone

AirLLM is a Python library that runs quantized GGUF models with extreme memory efficiency. Install it on a Python server, point CODI at it over Wi-Fi, and your phone becomes a local inference backend.

01
Install AirLLM

On the machine or device you want to run inference on (can be your phone via Termux, a Raspberry Pi, or a second PC):

pip install airllm

02
Create a server script

AirLLM exposes an OpenAI-compatible HTTP server. Create llm_server.py and run it — CODI connects to it automatically when MODE is set to air.

03
Set the URL in config.py

Find the LAN IP of the device running AirLLM (e.g. 192.168.1.42) and set:

AIR_LLM_URL = "http://192.168.1.42:8080"

04
Switch CODI to air mode

In the CODI CLI type /mode air, or set MODE = "air" in config.py. In hybrid mode, CODI falls back to AirLLM automatically when Ollama is offline.

Recommended models for AirLLM: Phi-3-mini-4k-instruct (Q4_K_M, ~2.2 GB) or Llama-3.2-3B-Instruct (Q4_K_M, ~1.8 GB). AirLLM's layer-by-layer loading means it runs on devices with as little as 3 GB RAM. See the AirLLM GitHub for the full model list and server setup guide. 
# Minimal AirLLM server (save as llm_server.py, run on inference device)
from airllm import AutoModel
from fastapi import FastAPI
import uvicorn

model = AutoModel.from_pretrained("microsoft/Phi-3-mini-4k-instruct")
app   = FastAPI()

@app.post("/v1/chat/completions")
async def chat(req: dict):
    msgs    = req.get("messages", [])
    prompt  = "\n".join(m["content"] for m in msgs)
    tokens  = model.tokenizer(prompt, return_tensors="pt")
    out     = model.generate(**tokens, max_new_tokens=512)
    text    = model.tokenizer.decode(out[0], skip_special_tokens=True)
    return {"choices": [{"message": {"content": text}}]}

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8080)
